## 二、MemoryStore 详解

### 2.1 文件结构

```
workspace/
├── USER.md              # 用户档案（偏好、习惯、个人信息）
├── SOUL.md              # Bot 人格（行为、语气、角色设定）
├── AGENTS.md            # 项目说明
└── memory/
    ├── MEMORY.md        # 长期记忆文件（已弃用，改用 ReMe）
    ├── history.jsonl    # 压缩后的对话历史（JSONL 格式）
    ├── .cursor          # 当前 history 游标（整数）
    └── .dream_cursor    # Dream 处理进度游标
```

### 2.2 history.jsonl 格式

每行一个 JSON 对象：

```json
{"cursor": 1, "timestamp": "2026-04-09 14:30", "content": "用户偏好使用 VS Code..."}
{"cursor": 2, "timestamp": "2026-04-09 15:45", "content": "讨论了项目架构决策..."}
```

**字段说明：**
- `cursor`: 自增整数，用于追踪处理进度
- `timestamp`: 压缩时间 `YYYY-MM-DD HH:MM`
- `content`: LLM 生成的压缩摘要或 `[RAW]` 原始消息

### 2.3 核心方法实现

#### append_history() - 追加历史记录

```python
def append_history(self, entry: str) -> int:
    """追加历史记录并返回游标值"""
    cursor = self._next_cursor()  # 从 .cursor 文件读取 +1
    ts = datetime.now().strftime("%Y-%m-%d %H:%M")
    record = {"cursor": cursor, "timestamp": ts, "content": strip_think(entry)}
    
    # 追加写入 JSONL
    with open(self.history_file, "a", encoding="utf-8") as f:
        f.write(json.dumps(record, ensure_ascii=False) + "\n")
    
    # 更新游标文件
    self._cursor_file.write_text(str(cursor), encoding="utf-8")
    return cursor
```

#### read_unprocessed_history() - 读取未处理的历史

```python
def read_unprocessed_history(self, since_cursor: int) -> list[dict]:
    """返回 cursor > since_cursor 的所有记录"""
```

#### compact_history() - 历史压缩

```python
def compact_history(self) -> None:
    """当记录数超过 max_history_entries 时删除最旧的"""
    if self.max_history_entries <= 0:
        return
    entries = self._read_entries()
    if len(entries) <= self.max_history_entries:
        return
    kept = entries[-self.max_history_entries:]  # 保留最新的
    self._write_entries(kept)
```

### 2.4 Legacy 迁移机制

支持从旧版 `HISTORY.md` 迁移到 `history.jsonl`：

## 三、Consolidator 压缩器详解

### 3.1 触发机制

Consolidator 是**轻量级 Token 触发**的记忆压缩器。

```
触发流程：
消息处理完成 → maybe_consolidate_by_tokens(session)
                          │
                          ▼
              estimate_session_prompt_tokens()
                          │
                          ▼
              是否超过 budget?
              (context_window - max_tokens - 1024 buffer)
                          │
                ┌─────────┴─────────┐
                │ 否                 │ 是
                ▼                   ▼
              返回             开始压缩循环
```

### 3.2 Token 预算计算

```python
budget = context_window_tokens - max_completion_tokens - _SAFETY_BUFFER(1024)
target = budget // 2  # 目标：使用预算的一半

# 示例：128K 上下文
# budget = 128000 - 4096 - 1024 = 122880
# target = 61440
```

### 3.3 压缩边界选择

```python
def pick_consolidation_boundary(self, session, tokens_to_remove):
    """选择安全的压缩边界（用户回合边界）"""
    start = session.last_consolidated
    
    for idx in range(start, len(session.messages)):
        message = session.messages[idx]
        
        # 在用户回合边界处记录候选
        if idx > start and message.get("role") == "user":
            last_boundary = (idx, removed_tokens)
            if removed_tokens >= tokens_to_remove:
                return last_boundary
        
        removed_tokens += estimate_message_tokens(message)
    
    return last_boundary
```

**关键设计：只在工作边界压缩**
- 避免在工具调用中途压缩
- 保证上下文完整性
- 防止丢失重要上下文

```python
async def archive_with_reme(self, messages: list[dict]) -> bool:
    """使用 ReMe 进行对话摘要，失败时降级"""
    
    if not messages:
        return False
    
    # 尝试 ReMe
    if self.reme_adapter:
        try:
            user_id = self._extract_user_name()  # 从 USER.md 提取用户名
            await self.reme_adapter.summarize_conversation(
                messages, 
                user_id=user_id
            )
            logger.info("Archived {} messages via ReMe for user '{}'", 
                       len(messages), user_id)
            return True
        except Exception as e:

    # 降级：使用 LLM 直接压缩
    return await self.archive(messages)
```

### 4.4 知识事实存储到 ReMe

```python
async def _store_facts_to_reme(self, analysis: str, history_text: str):
    """提取知识事实并存入 ReMe"""
    
    if not self.reme_adapter or not self.reme_adapter.is_healthy():
        return
    
    # 从分析结果中提取非 USER/SOUL 的事实
    facts = []
    for line in analysis.split("\n"):
        line = line.strip()
        if not line:
            continue
        # 跳过 USER/SOUL/SKIP 标记的行
        if line.startswith("[USER]") or line.startswith("[SOUL]") or line.startswith("[SKIP]"):
            continue
        facts.append(line)
    
    # 存入 ReMe
    user_id = self._extract_user_name()
    for fact in facts:
        content = f"[knowledge] {fact}"
        await self.reme_adapter.add_memory(content, user_id=user_id)
```

### 5.2 断路器保护机制

```python
class RemeMemoryAdapter:
    # 断路器阈值
    MAX_FAILURES = 3           # 失败3次后断开
    RECOVERY_TIMEOUT = 60      # 60秒后尝试恢复
    
    # 死循环保护
    MIN_RETRIEVAL_INTERVAL = 5.0      # 最小检索间隔（秒）
    MAX_RETRIEVALS_PER_MINUTE = 10    # 每分钟最大检索次数
    
    def _check_circuit(self) -> bool:
        """检查断路器是否允许操作"""
        if not self._circuit_open:
            return True
        
        # 检查是否超过恢复超时
        elapsed = time.time() - self._last_failure_time
        if elapsed >= self.RECOVERY_TIMEOUT:
            logger.info("ReMe circuit breaker attempting recovery")
            return True
        
        return False
    
    def _record_failure(self, error: Exception, operation: str):
        """记录失败并更新断路器状态"""
        self._failure_count += 1
        self._last_error = f"{operation}: {type(error).__name__}: {error}"
        self._last_failure_time = time.time()
        
        if self._failure_count >= self.MAX_FAILURES:
            self._circuit_open = True
            self._healthy = False
            logger.error(f"ReMe circuit breaker OPENED after {self._failure_count} failures")
```

### 5.3 死循环保护

```python
def _check_dead_loop(self) -> tuple[bool, str]:
    """检查潜在的死循环条件"""
    now = time.time()
    
    # 1. 检测递归检索
    if self._retrieval_in_progress:
        return True, "Recursive retrieval detected"
    
    # 2. 检查最小间隔
    elapsed = now - self._last_retrieval_time
    if elapsed < self.MIN_RETRIEVAL_INTERVAL:
        return True, f"Too soon since last retrieval ({elapsed:.1f}s)"
    
    # 3. 检查每分钟最大检索次数
    self._retrieval_times = [t for t in self._retrieval_times if now - t < 60.0]
    if len(self._retrieval_times) >= self.MAX_RETRIEVALS_PER_MINUTE:
        # 强制断开断路器
        self._circuit_open = True
        self._healthy = False
        return True, "Too many retrievals per minute"
    
    return False, ""
```

### 5.4 核心操作实现

#### summarize_conversation() - 对话摘要

```python
async def summarize_conversation(
    self,
    messages: list[dict],
    user_id: str | None = None,
    task_name: str | None = None,
) -> list[MemoryNode]:
    """从对话中提取记忆"""
    
    self._ensure_started()
    if not self._check_circuit():
        return []
    
    # 转换消息格式
    formatted_messages = self._format_messages_for_reme(messages)
    
    kwargs = {
        "user_name": user_id or self._default_user_id,
        "retrieve_top_k": self.config.retrieve_top_k,
    }
    
    if task_name and self.config.enable_procedural_memory:
        kwargs["task_name"] = task_name
    
    result = await self._reme.summarize_memory(
        messages=formatted_messages,
        **kwargs
    )
    
    self._record_success()
    return result
```

### 5.5 消息格式转换

```python
def _format_messages_for_reme(self, messages: list[dict]) -> list[dict]:
    """转换 nanobot 消息格式为 ReMe 格式"""
    
    formatted = []
    for msg in messages:
        role = msg.get("role", "")
        if role not in ("user", "assistant"):
            continue
        
        content = msg.get("content", "")
        if isinstance(content, list):
            # 处理多模态内容
            text_parts = [
                block.get("text", "")
                for block in content
                if block.get("type") == "text"
            ]
            content = "\n".join(text_parts)
        
        if not content:
            continue
        
        formatted.append({
            "role": role,
            "content": content,
            "time_created": msg.get("timestamp", datetime.now().strftime("%Y-%m-%d %H:%M:%S")),
        })
    
    return formatted
```

### 6.1 用户名提取逻辑

```python
def _extract_user_name(self) -> str:
    """从 USER.md 提取用户名"""
    try:
        user_content = self.store.read_user()
        if not user_content:
            return "default_user"
        
        import re
        
        # 匹配中文格式: **名字：** 张烽林
        match = re.search(r"\*\*名字[：:]\*\*\s*(\S+)", user_content)
        if match:
            return match.group(1)
        
        # 匹配英文格式: **Name**: John
        match = re.search(r"\*\*Name\*\*[：:]\s*(\S+)", user_content, re.IGNORECASE)
        if match:
            return match.group(1)
    except Exception:
        pass
    
    return "default_user"
```

## 七、GitStore 版本控制

### 7.1 Git 仓库管理

```python
class GitStore:
    """为记忆文件提供 Git 版本控制"""
    
    def __init__(self, workspace: Path, tracked_files: list[str]):
        self._workspace = workspace
        self._tracked_files = tracked_files  # ["SOUL.md", "USER.md"]
    
    def is_initialized(self) -> bool:
        return (self._workspace / ".git").is_dir()
    
    def init(self) -> bool:
        """初始化 Git 仓库"""
        porcelain.init(str(self._workspace))
        
        # 创建 .gitignore
        gitignore.write_text(self._build_gitignore())
        
        # 初始提交
        porcelain.add(str(self._workspace), paths=[".gitignore"] + self._tracked_files)
        porcelain.commit(str(self._workspace), message=b"init: nanobot memory store")
```

### 7.2 自动提交

```python
def auto_commit(self, message: str) -> str | None:
    """自动提交变更"""
    if not self.is_initialized():
        return None
    
    # 检查是否有变更
    st = porcelain.status(str(self._workspace))
    if not st.unstaged and not any(st.staged.values()):
        return None
    
    # 暂存并提交
    porcelain.add(str(self._workspace), paths=self._tracked_files)
    sha_bytes = porcelain.commit(
        str(self._workspace),
        message=message.encode("utf-8"),
        author=b"nanobot <nanobot@dream>",
    )
    
    return sha_bytes.hex()[:8]  # 返回短 SHA
```

### 7.4 回滚功能

```python
def revert(self, commit: str) -> str | None:
    """回滚到指定提交的父提交"""
    full_sha = self._resolve_sha(commit)
    
    with Repo(str(self._workspace)) as repo:
        commit_obj = repo[full_sha]
        parent_obj = repo[commit_obj.parents[0]]
        
        # 从父提交恢复文件
        for filepath in self._tracked_files:
            content = self._read_blob_from_tree(repo, parent_obj.tree, filepath)
            if content:
                (self._workspace / filepath).write_text(content)
    
    # 创建回滚提交
    return self.auto_commit(f"revert: undo {commit}")
```

```python
        self._get_user_name = get_user_name  # 注入用户名获取函数
```

## 十一、关键设计决策

### 11.1 为什么 MEMORY.md 不再注入上下文？

**原因：**
1. **Token 效率**: 自动注入会消耗大量 token
2. **LLM 自主性**: 让 LLM 决定何时需要检索记忆
3. **语义检索**: ReMe 的语义检索比关键词匹配更精准

**替代方案：**
- `retrieve_memory` 工具让 LLM 主动检索
- 仅在用户提到"之前"、"上次"等时触发

### 11.2 为什么需要 Consolidator 和 Dream 两层压缩？

**Consolidator (轻量级):**
- Token 触发，实时响应
- 快速压缩，防止上下文溢出
- 输出到 history.jsonl

**Dream (重量级):**
- 定时执行，深度分析
- 提取结构化信息更新档案文件
- Git 版本控制

### 11.3 断路器保护的必要性

**ReMe 操作特点：**
- 检索耗时 30-60 秒（多阶段 LLM 调用）
- 可能因网络/配置问题失败
- 失败时不应阻塞主流程

**断路器策略：**
- 失败 3 次后断开
- 60 秒后尝试恢复
- 降级到文件型记忆

### 11.4 用户归因的价值

**应用场景：**
- 多用户场景区分记忆归属
- 个性化记忆检索
- 记忆所有权追踪

**实现方式：**
- 从 USER.md 自动提取用户名
- 所有记忆操作携带 user_id 参数
- ReMe 按 user_id 组织记忆
